# T+W Organigram & Resource Directory for Badger

**Purpose:** This is the living map of every person in the organization. Badger uses this for intelligent scheduling, workload balancing, personalized Slack communication, and skills matching. Update this file (or the in-app form) whenever people join, change roles, go on leave, or preferences shift. Keep it accurate — it's the foundation of fair and effective resourcing.

## Format
Each person gets a clear block. Use consistent fields. Add/remove as needed. For availability, note recurring patterns (e.g., "Tue/Thu part-time") or specific dates in Notes.

## How Badger Uses This
- **Matching:** When a task requires "strong client workshop facilitation", Badger prioritizes people with that skill who have capacity.
- **Capacity Math:** Daily hours allocated to a person across all active tasks ≤ their Default Daily Productive Capacity (with buffer for meetings/unplanned).
- **Communication:** Vic addresses them by first name or role naturally. Uses correct Slack handle for DMs or @mentions in channels.
- **Workload Views:** Dashboard shows % utilized per person per week. Flags anyone >85-90% consistently or sudden spikes.
- **Holidays/Changes:** When Notes updated with leave, Badger automatically de-allocates or warns on affected dates.

## Maintenance Ritual
- PMs or Ops: Review/update this file or in-app form at least weekly or when headcount changes.
- Before big new project kickoff: Badger (or you) reviews current utilization and flags if new work would break balance.
- Quarterly: Revisit capacities and skills as people grow or roles evolve.

**Pro Tip:** Keep this honest and updated. Over-optimistic capacity numbers lead to burnout and missed deadlines. Under-estimating leads to under-utilization and lost revenue. Badger thrives on accurate data here.
